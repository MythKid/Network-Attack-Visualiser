# Runtime dependencies for the Network Attack Visualiser backend.
#
# Pip-compatible and intentionally minimal — no Poetry/uv/Pipenv/Conda. Only
# dependencies genuinely required by the current phase are listed. Exact pins on
# these direct dependencies reduce unintended dependency drift; their transitive
# dependencies are resolved by pip and are not locked here. Bump deliberately
# (e.g. via a reviewed PR), not implicitly.
#
# Phase 1 (backend skeleton): the FastAPI application, the ASGI server and the
# typed settings stack. Plain `uvicorn` (no [standard] extras) is sufficient to
# serve the app; performance extras are added only in the phases that need them.
#
# Phase 3 (alert pipeline): `websockets` provides the server-side WebSocket
# protocol implementation for `WS /api/v1/ws/alerts`. Plain uvicorn ships no
# WebSocket implementation, so without this the server rejects every upgrade
# (Starlette's TestClient fakes WebSockets in-process and cannot catch that).
# BSD-3-Clause; pure Python; uvicorn's default `auto` protocol selects it.
